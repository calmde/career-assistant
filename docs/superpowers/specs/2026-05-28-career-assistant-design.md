# 四步一站式求职助手 — 设计文档

## 背景

现有项目是一个简历-职位匹配系统（爬虫 + 关键词匹配 + Web/GUI展示）。目标是将它升级为面向**大学毕业生**的全流程求职助手，围绕四个步骤：解析岗位 → 评估匹配度 → 优化简历 → 弥补技能差距。

## 关键决策

| 决策点 | 选择 |
|---|---|
| 目标用户 | 即将毕业的大学生 |
| 核心定位 | 全流程均衡（四步均等投入） |
| 交互形式 | Flask Web 应用 |
| LLM 策略 | 混合模式：本地规则做匹配，LLM API 做简历优化和技能建议 |

## 架构概览

```
用户上传简历 ──→ ① 岗位解析 ──→ ② 匹配评估 ──→ ③ 简历优化 ──→ ④ 技能弥补
     │               │                │               │               │
     │          爬虫抓取JD       本地规则+LLM      LLM改写        LLM生成
     │          结构化提取       综合打分          关键词调整     学习路径
     │          需求标签化       匹配报告          量化描述       项目推荐
     v
  PostgreSQL/MySQL
  (迁移自SQLite)
```

## 模块设计

### 模块1：岗位解析器 `job_parser.py`

**职责**: 将非结构化岗位文本转换为结构化需求标签。

**输入**: 岗位描述文本（来自爬虫）
**输出**: 结构化需求字典

**提取维度**:
- 硬技能（200+词表 + 别名归一化，如 React=React.js=reactjs）
- 经验要求（中文数字"三年以上"、应届/不限/实习）
- 学历要求（保持现有 DEGREE_RANK）
- 岗位类别（后端/前端/算法/数据/测试/运维/产品）
- 行业领域（金融/教育/电商/游戏/企业服务）
- 软技能（沟通/团队协作/项目管理/抗压）
- 福利标签（远程/双休/六险一金/股票期权）

**技术方案**: jieba 分词 + 关键词词表 + 正则表达式 + 岗位类别分类器

### 模块2：匹配引擎 `matcher.py`

**职责**: 简历与岗位多维打分 + 差距分析。

**输入**: 简历结构化数据 + 岗位结构化数据
**输出**: 匹配报告（分数、匹配项、缺失项、建议）

**评分权重**（两层）:

第一层 — 本地规则评分:
- 技能覆盖度 × 40%
- 经验匹配度 × 20%
- 学历匹配度 × 20%
- 岗位类别匹配 × 10%
- 行业经验匹配 × 10%

第二层 — LLM 语义复核（可选，对 top-N 岗位）:
- 段落级语义相似度
- 发现隐性匹配（用户忘写但实际具备的技能）
- 输出可读的匹配报告

**实现**: 基于现有 `analyze_match.py` 重构，拆分为规则评分 + LLM复核两个独立函数。

### 模块3：简历优化器 `resume_optimizer.py`

**职责**: 针对目标岗位生成简历改进建议。

**功能**:
- 关键词布局建议（JD高频词在简历中的最佳位置）
- 项目描述量化改写（"做了XX" → "用X技术完成XX，达到Y指标"）
- 缺失技能包装策略（相关经验如何表述来弥补）
- 多版本自我评价生成（针对不同岗位方向）
- ATS 友好度检测（关键词密度检查）

**技术方案**: LLM API（DeepSeek/通义千问），带结构化 Prompt 模板。

### 模块4：技能导航 `skill_navigator.py`

**职责**: 针对缺失技能生成可执行的学习计划。

**功能**:
- 技能重要性分级（核心必备 vs 加分项）
- 缺口等级评估：边缘缺失(1-2项) → 速成路径；核心缺失(3-5项) → 系统计划
- 学习资源推荐（课程、书籍、练手项目）
- 替代岗位推荐（如果缺口太大，推荐更匹配的岗位方向）

**技术方案**: LLM API + 本地课程/项目模板库。

### 模块5：Web 界面 `app.py` + `templates/`

**页面结构**（多页面 Flask）:

| 路由 | 页面 | 功能 |
|---|---|---|
| `/` | 首页 | 上传简历 + 城市/岗位偏好设置 |
| `/jobs` | 岗位浏览 | 爬取结果浏览、筛选、搜索 |
| `/match/<job_id>` | 匹配详情 | 简历vs岗位评分、亮点/风险、匹配报告 |
| `/optimize/<job_id>` | 简历优化 | 针对该岗位的改写建议、关键词调整 |
| `/skill-plan/<job_id>` | 技能计划 | 缺失技能清单 + 学习路径 + 替代岗位推荐 |

### 模块6：管线编排 `pipeline.py`

**职责**: 解耦后的流程编排器，可独立调用各步骤。

**流程**:
1. 解析简历 → 提取关键词
2. 爬取岗位（多源并行）
3. 批量匹配评估
4. 对 top-N 岗位生成优化建议
5. 对 top-N 岗位生成技能计划

## LLM 集成设计

**调用策略**:
- 匹配评估 LLM 复核：仅对规则评分 top-10 岗位调用（每天最多 10 次）
- 简历优化：用户手动触发（每次消耗 1 次调用）
- 技能计划：用户手动触发（每次消耗 1 次调用）

**API 配置**:
- 环境变量 `LLM_API_KEY`、`LLM_API_BASE`、`LLM_MODEL`
- 支持 OpenAI 兼容接口（DeepSeek、通义千问等）
- LLM 不可用时降级为纯本地模式

**Safety**:
- Prompt 模板硬编码在代码中，不拼接用户输入
- API Key 不记录日志
- 结果缓存（同岗位+同简历不重复调用）

## 验证方式

1. **岗位解析**: 输入 10 条真实 JD，人工检查结构化提取准确率
2. **匹配评估**: 用已知匹配/不匹配的简历-岗位对，验证评分区分度
3. **简历优化**: 模拟真实场景，检查生成的建议是否具体可操作
4. **技能计划**: 验证推荐的学习路径步骤是否合理连续
5. **端到端**: 上传一份真实简历，走通四步全流程，检查每步输出
6. **降级测试**: 断开 LLM API，确认系统正常降级到纯本地模式
